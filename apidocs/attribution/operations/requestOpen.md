Attribution API requestOpen Request Open The network request to v1/open should occur when you are successfully notified the app was brought to the foreground. Branch defines an app lifecycle as a session of when your application is launched (cold start, or if it was present in the iOS or Android task manager), and comes to the device’s foreground. Attribution API branch_key The Branch Key of the originating app obtained in your [Account Settings](https://help.branch.io/using-branch/docs/profile-settings) branch_secret The Branch secret of the originating app obtained in your [Account Settings](https://help.branch.io/using-branch/docs/profile-settings) server_to_server Must be `true` for server-to-server attribution requests. Required. app_version Your app's version model Model of the device. os System OS. Only accepts Android or iOS os_version The version of the device's OS (e.g. “23” for Android or “9.1.2” for iOS respectively. Will accept any value, preferred default to “NA”. This cannot be blank.) user_agent The user agent of the device is_hardware_id_real Set to false if the hardware_id property cannot be populated, otherwise default to true ad_tracking_enabled A value used to determine whether the IDFA or GAID can be used for advertising purposes. If Limit Ad Tracking should be enabled, do not include in the request. hardware_id In iOS, populate with the IDFA if it's available, otherwise fallback to IDFV. On Android, populate with the ANDROID_ID (hardware id). If no device ID is available, do not include in request. hardware_id_type Valid for iOS ONLY. Value confirms whether the iOS ID in the hardware_id property is an "idfa" or "vendor_id" (do not include in request, if hardware_id is not populated) ios_vendor_id iOS ONLY The IDFV if it's available, otherwise do not include in request. google_advertising_id Valid for ANDROID ONLY. The Google Advertising ID if it's available, otherwise do not include in request advertising_ids ANDROID ONLY The set of Advertising IDs if it's available, otherwise do not include in request. Can include identifiers like OAID local_ip This is the device's local IP address, if one is available. This is incremental to the falserequired IP header. An optional but highly recommended data metadata that improves attribution accuracy. screen_height Height of the device’s screen. Only include in the app's first session, if it's available. apple_attribution_token Valid for iOS ONLY Attribution Token used for Apple Search Ads . used to call the Apple Search Ads Attribution API. The token has a 24 hr TTL, so send to Branch within that time in order to use it for attribution. first_session Identifies if this is the first time the user opened the app. If true, Branch will check if we saw the user download the app before, and mark the session as a reinstall if yes, and an install if no. If this value was set to false, Branch will count the session as an open. server_to_server_identity Setting to true enables the ability to set an randomized_bundle_token property in v1/open requests and returns a value in the response. Do not set this property in the request, if this feature will not be used. randomized_bundle_token If server_to_server_identity is true, Branch returns a new identity value in any request that does not set randomized_bundle_token. After the first request, this property must be hardcoded with the randomized_bundle_token received in the first network response. Failure to return this value in subsequent requests severely limits the ability to connect different events for that user. identity If server_to_server_identity is true, this property can be used to associate a custom user ID with Branch's anonymous identity.
